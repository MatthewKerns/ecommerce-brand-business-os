# User Journey: TikTok Studio

## Overview

TikTok Studio is the **content automation command center** for the 4-channel elemental system (Fire, Earth, Water, Air). Each channel targets a different emotional frequency and serves a distinct strategic purpose. This tab is where content is ideated, generated, reviewed, scheduled, and analyzed â€” across all connected TikTok accounts.

This product is built for the **top 1% of e-commerce sellers** who already have TikTok presence and want to systematize and scale their content operation. Some arrive with 1-4 existing TikTok channels. Others need to spin up new channels quickly. The system supports both workflows and uses every available context (brand guide, existing content, blog connections, email campaigns) to accelerate setup.

## Data Model

The TikTok Studio produces and consumes these core data structures:

```typescript
interface TikTokChannel {
  id: string;
  workspaceId: string;
  element: 'fire' | 'earth' | 'water' | 'air';
  tiktokAccountId: string;
  username: string;
  displayName: string;
  followerCount: number;
  connectionStatus: 'connected' | 'disconnected' | 'expired';
  automationEnabled: boolean;
  postingSchedule: PostingSchedule;
  agentConfig: ChannelAgentConfig;
  createdAt: Date;
  updatedAt: Date;
}

interface ChannelAgentConfig {
  channelId: string;
  persona: string; // "The Hype Commander", "The Community Elder", etc.
  voiceProfile: {
    tone: string[];
    phrases: string[];
    avoidPhrases: string[];
  };
  contentPillars: ContentPillar[];
  productIntegrationLevel: 'every-video' | 'subtle' | 'rare';
  metricsTargets: {
    views: number;
    saveRate: number;
    shareRate: number;
    commentTarget: number;
  };
}

interface TikTokDraft {
  id: string;
  channelId: string;
  status: 'generating' | 'draft' | 'in_review' | 'approved' | 'scheduled' | 'published' | 'failed';
  script: {
    hook: string;
    build: string;
    payoff: string;
    cta: string;
  };
  caption: string;
  hashtags: string[];
  productTieIn?: string;
  contentPillar: string;
  brandCheckResult: BrandCheckResult;
  scheduledAt?: Date;
  publishedAt?: Date;
  metrics?: ContentMetrics;
  generatedBy: 'ai' | 'manual';
  reviewedBy?: string;
  createdAt: Date;
  updatedAt: Date;
}

interface BrandCheckResult {
  passed: boolean;
  voiceMatch: boolean;
  languageCompliant: boolean;
  violations: {
    type: 'avoided_word' | 'off_brand_tone' | 'missing_cta';
    detail: string;
    suggestion: string;
  }[];
}

interface PostingSchedule {
  channelId: string;
  weeklySlots: {
    dayOfWeek: number; // 0-6
    time: string; // "10:00"
    contentPillar?: string;
  }[];
  timezone: string;
}
```
