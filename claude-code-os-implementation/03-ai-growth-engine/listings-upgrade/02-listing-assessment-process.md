# Listing Assessment & Validation Process

## Framework Origin

This assessment process is adapted from the Titan challenge methodology for systematically evaluating and improving Amazon listings. The goal is to move from subjective "I think this looks good" to objective, scored evaluation that reveals specific improvement areas.

## Part 6: Validation Process

After creating upgraded content, validate before going live.

### Pre-Launch Validation Checklist

- [ ] **Brand Voice Check**: Read all copy aloud - does it sound like Infinity Vault?
- [ ] **Commodity Language Scan**: Search for "storage," "box," "container," "cheap," "basic" - remove all
- [ ] **Emotion Check**: Every bullet has at least one emotional word or scenario
- [ ] **Differentiation Check**: Could a competitor copy this with their brand name? If yes, rewrite
- [ ] **Avatar Check (Blake)**: Would a tournament player feel this is for them?
- [ ] **Avatar Check (Chris)**: Would a collector feel this protects their investment?
- [ ] **Avatar Check (Pat)**: Would a parent feel confident buying this as a gift?
- [ ] **Keyword Check**: All target keywords present in title and/or bullets
- [ ] **Warranty Prominence**: Lifetime warranty mentioned in at least 3 places
- [ ] **Visual Alignment**: Image briefs match verbal messaging direction

### A/B Testing Protocol

| Test | Variable | Duration | Primary Metric | Secondary Metric |
|------|----------|----------|----------------|-----------------|
| 1 | Main image (warranty badge vs. no badge) | 14 days | CTR | CVR |
| 2 | Title variation (A vs. B) | 14 days | CTR | Impressions |
| 3 | Bullet points (benefit-first vs. current) | 14 days | CVR | Add-to-cart rate |

**Rules**:
- One variable at a time
- Minimum 7 days, prefer 14 days
- Statistical significance before declaring winner
- Document every test and result
